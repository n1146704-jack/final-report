前言

Vue 3 引入了 Composition API,這是一個全新的編寫元件邏輯的方式。相較於 Options API,Composition API 提供了更好的邏輯重用性和程式碼組織方式。本文將深入探討 Composition API 的核心概念與實際應用。

💡

提示

如果你還不熟悉 Vue 3 的基礎知識,建議先閱讀官方文檔中的入門教程。

什麼是 Composition API?

Composition API 是 Vue 3 的核心特性之一,它允許我們使用函式的方式來組織元件的邏輯。透過 setup() 函式,我們可以更靈活地組合和重用邏輯。

Options API

  • ✓ 容易學習和理解
  • ✓ 適合小型專案
  • ✗ 邏輯分散在不同選項中
  • ✗ 重用邏輯較困難

Composition API

  • ✓ 邏輯集中,易於維護
  • ✓ 更好的 TypeScript 支援
  • ✓ 容易重用邏輯
  • ✓ 適合大型專案

核心概念

1. setup() 函式

setup() 是 Composition API 的入口點,在元件建立之前執行。

JavaScript 
                                
                                    import { ref } from 'vue';

                                    export default {
                                      setup() {
                                        const count = ref(0);
                                        const increment = () => count.value++;
                                    
                                        return { count, increment };
                                      }
                                    }

2. ref 和 reactive

ref 用於建立響應式的基本型別資料,而 reactive 則用於物件。

JavaScript 
                                
                                    import { ref, reactive } from 'vue';

                                    const count = ref(0);
                                    const state = reactive({
                                      name: 'Vue 3',
                                      version: '3.0'
                                    });
⚠️

注意

使用 ref 時,需要透過 .value 來存取和修改值,但在模板中會自動解包。

3. computed 和 watch

計算屬性和監聽器在 Composition API 中也變得更加靈活。

JavaScript 
                                
                                    import { ref, computed, watch } from 'vue';

                                    const count = ref(0);
                                    const doubleCount = computed(() => count.value * 2);

                                    watch(count, (newVal, oldVal) => {
                                      console.log(`count 從 ${oldVal} 變更為 ${newVal}`);
                                    });

實際應用範例

讓我們來看一個實際的待辦事項應用範例:

JavaScript 
                                
                                    import { ref, computed } from 'vue';

                                    export default {
                                      setup() {
                                        const todos = ref([
                                          { id: 1, text: '學習 Vue 3', done: false },
                                          { id: 2, text: '使用 Composition API', done: false }
                                        ]);
                                    
                                        const newTodo = ref('');
                                    
                                        const addTodo = () => {
                                          if (newTodo.value.trim()) {
                                            todos.value.push({
                                              id: Date.now(),
                                              text: newTodo.value,
                                              done: false
                                            });
                                            newTodo.value = '';
                                          }
                                        };
                                    
                                        const remainingCount = computed(() => {
                                          return todos.value.filter(t => !t.done).length;
                                        });
                                    
                                        return {
                                          todos,
                                          newTodo,
                                          addTodo,
                                          remainingCount
                                        };
                                      }
                                    }

優勢與最佳實踐

📝

更好的組織

相關邏輯可以放在一起,而不是分散在不同的選項中

🔄

邏輯重用

透過 composable functions 輕鬆重用邏輯

📘

TypeScript 支援

更好的型別推斷和 IDE 支援

⚡

更好的效能

沒有 Options API 的額外開銷

最佳實踐建議

  • ✅
    使用 composables 抽取可重用邏輯

    將相關的狀態和方法組合成可重用的函式

  • ✅
    保持 setup() 簡潔

    將複雜邏輯拆分到獨立的 composables 中

  • ✅
    善用 TypeScript

    利用 TypeScript 提供更好的型別安全

  • ✅
    遵循命名慣例

    使用 use 開頭命名 composables,如 useCounter、useUser

結論

Composition API 為 Vue 3 帶來了更強大的邏輯組織能力。雖然初學時可能需要一些時間適應,但一旦掌握後,你會發現它在處理複雜邏輯時的優勢。建議在新專案中優先考慮使用 Composition API,享受它帶來的開發體驗提升。
